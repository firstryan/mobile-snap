MobileSnap 📸

Otomasi Pixel-Precise App Store & Play Store Screenshots dari Server Lokal secara Dual-Vendor menggunakan NPX.

Node.js / NPX
Playwright
iOS & Android

🏗️ Arsitektur Aliran Kerja

MobileSnap dibangun menggunakan Commander untuk antarmuka CLI yang kaya, dan Playwright Node API untuk mengontrol Chromium secara berurutan pada berbagai dimensi.

1

CLI Parser

Membaca parameter URL, rute (paths), platform target, dan folder keluaran.

2

Browser Engine

Inisiasi Chromium secara terisolasi dan asinkronus.

3

Viewport Emulation

Meniru dimensi layar Retina (iOS) & Android secara dinamis.

4

Hydration Check

Menunggu aktivitas jaringan idle agar rendering JavaScript selesai.

🚀 Cara Penggunaan Instan (NPX)

Anda tidak perlu melakukan penyiapan manual. Cukup gunakan npx untuk menjalankan MobileSnap.

PowerShell - Setup
# 1. Jalankan instan menggunakan npx (ganti port sesuai server lokal Anda)
PS>npx mobile-snap --url http://localhost:4321

# 2. Opsional: Unduh browser binaries jika Playwright belum dikonfigurasi di sistem
PS>npx playwright install chromium

⚡ CLI Command Builder (NPM / NPX)

Gunakan builder interaktif ini untuk membuat dan menyalin perintah sesuai kebutuhan pengembangan Anda:

npx mobile-snap --url http://localhost:4321 --paths "/, /scan, /profile" --output mobilesnap_output

⚙️ Parameter & Opsi CLI

Parameter Alias Deskripsi Bawaan (Default)
--url -u (Wajib) URL server lokal yang sedang berjalan (contoh: Astro). -
--paths -p Daftar rute halaman yang dipisahkan dengan tanda koma. "/"
--platform -l Platform yang ingin ditangkap: "ios", "android", atau "both". "ios"
--output -o Folder tujuan untuk menaruh berkas gambar hasil tangkapan. "mobilesnap_output"

🛡️ Ketahanan Sistem & Fitur Khusus

💡
Skema Protokol URL & Auto-Normalisasi

MobileSnap secara otomatis mendeteksi jika URL tidak memiliki protokol (seperti localhost:3000) dan menambahkan http:// secara dinamis.

🌐
User-Agent Dinamis per Platform

Ketika platform Android berjalan, mesin otomatisasi mengemulasi Agen Pengguna Android 13 (Pixel 7). Sedangkan untuk platform iOS, digunakan Agen Pengguna iPhone. Ini menjamin server memberikan respon tampilan yang tepat sesuai target vendor.

📁
Skala Resolusi Retina (DPI 3x)

Semua screenshot ditangkap dengan pengaturan deviceScaleFactor: 3 untuk menjaga kualitas output tetap tajam untuk standar unggahan store resmi.
