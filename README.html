🏗️ Workflow Architecture

MobileSnap is built using Commander for a rich CLI interface, and the Playwright Node API to control Chromium sequentially across various dimensions.

1

CLI Parser

Parses URL parameters, routes (paths), target platforms, and the output directory.

2

Browser Engine

Launches Chromium in an isolated and asynchronous instance.

3

Viewport Emulation

Dynamically emulates Retina (iOS) & Android display dimensions.

4

Hydration Check

Waits for network idle state to ensure JavaScript rendering is fully complete.

🚀 Instant Usage (NPX)

You do not need any manual setup. Simply use npx to run MobileSnap instantly.

PowerShell - Setup
# 1. Run instantly using npx (change port according to your local server)
PS>npx mobile-snap --url http://localhost:4321

# 2. Optional: Download browser binaries if Playwright is not yet configured in the system
PS>npx playwright install chromium

⚡ CLI Command Builder (NPM / NPX)

Use this interactive builder to generate and copy the command that fits your development needs:

npx mobile-snap --url http://localhost:4321

⚙️ CLI Parameters & Options

Option Alias Description Default
--url -u (Required) URL of the local development server (e.g., Astro). -
--paths -p Comma-separated list of routes to capture. "/"
--platform -l Target platform to capture: "ios", "android", or "both". "ios"
--output -o Destination directory to save the screenshot files. "mobilesnap_output"
--crawl -c Automatically discover and screenshot all internal links starting from the home page. false
--detect-pages -d Scan the local project directories (Astro/Next.js) for static routes. false
--email - Email / Username for automatic authentication if the page requires login. -
--password - Password for automatic authentication if the page requires login (masked input). -
--login-path - URL route path to the login page. "/login.html"
--html - Automatically append .html extension to detected static routes. false
--mockup -m Wrap screenshots in a beautiful device mockup frame (iPhone/Android) with status bars and transparent drop shadows. false

🛡️ System Resilience & Special Features

💡
URL Protocol & Auto-Normalization

MobileSnap automatically detects if the URL lacks a protocol prefix (e.g., localhost:3000) and prepends http:// dynamically.

🌐
Dynamic User-Agents per Platform

When capturing for Android, the system emulates Android 13 (Pixel 7) user-agent. For iOS, it emulates iPhone. This guarantees that your server serves the appropriate layout for each specific vendor.

📁
Retina Resolution Scale (DPI 3x)

All screenshots are captured with deviceScaleFactor: 3 to ensure the output remains ultra-sharp, complying with official store upload standards.
